Phase

Phases represent tags that tell the scheduler when to schedule a set of systems.

Properties​

PreStartup​

</>
Phase.PreStartup: Phase

Runs before the Startup Phase.

Startup​

</>
Phase.Startup: Phase

This Phase will run once, the first time the Scheduler is ran, before any other Phases are ran.

PostStartup​

</>
Phase.PostStartup: Phase

Runs after the Startup phase.

PreRender​

</>
Phase.PreRender: Phase

Runs on the RunService.PreRender Event.

PreAnimation​

</>
Phase.PreAnimation: Phase

Runs on the RunService.PreAnimation Event.

PreSimulation​

</>
Phase.PreSimulation: Phase

Runs on the RunService.PreSimulation Event.

PostSimulation​

</>
Phase.PostSimulation: Phase

Runs on the RunService.PostSimulation Event.

First​

</>
Phase.First: Phase

Will always run first on the RunService.Heartbeat Event.

PreUpdate​

</>
Phase.PreUpdate: Phase

Runs before the Update Phase.

Update​

</>
Phase.Update: Phase

Runs on the RunService.Heartbeat Event. This is the default Phase systems will run on.

PostUpdate​

</>
Phase.PostUpdate: Phase

Runs after the Update Phase.

Last​

</>
Phase.Last: Phase

Will always run last on the RunService.Heartbeat Event.
